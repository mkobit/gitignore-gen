# Objective
Enhance `gitignore_gen.py` to support local file sources (directories and archives), automatically use `GITHUB_TOKEN` for authentication when available, refine file headers for better append-mode support, and document out-of-scope multi-source/`gh` CLI delegation ideas as TODOs.

# Key Files & Context
- `gitignore_gen.py`

# Implementation Steps
1.  **Auth & Headers**:
    - Update `_get_archive` to check for `os.environ.get("GITHUB_TOKEN")`. If present and the URL contains `github.com`, inject an `Authorization: token ...` header.
    - Update the default `file-header-template` to start with a newline (e.g., `\n# Generated by...`) so appending via `>>` has breathing room, or add a `--append` flag that handles this smartly. We'll simply tweak the default template and add an "Appending" example to the docs.
2.  **Local Sources**:
    - Add `--local-dir` and `--local-archive` to the "Repository source" argument group.
    - Introduce a lightweight `TemplateSource` interface (or just a generic `TemplateMember` object) to decouple the matching and extraction logic from `tarfile`.
    - `TemplateMember` will have `.name` (relative path) and a method to read its contents.
    - If `--local-dir` is provided, `pathlib.Path.rglob` is used to build the members list.
    - If `--local-archive` is provided, it opens the local tarball instead of downloading.
3.  **Documentation & TODOs**:
    - Add a `# TODO (Out of scope):` block at the top of the file mentioning:
      - Full multi-source aggregation in a single run (e.g., `--source name=internal,...`).
      - `.netrc` support for internal repos.
      - Delegating download entirely to the `gh api` CLI tool if installed.
    - Add an example of appending to a file in the docstring (`python3 - generate macOS >> .gitignore`).
4.  **Verification**:
    - Run the strict `ruff` and `ty` commands.
    - Test generating from a local directory (e.g., clone the repo or create a mock directory).
